# Наблюдаемость и метрики

**Обновлено:** 2026-04-17

Основной переключатель: `METRICS_TXT_ENABLED` (дефолт `1`).  
При включении все данные пишутся в **`bot/data/metrics/*.txt`** (каталог в `.gitignore`).  
При `METRICS_TXT_ENABLED=0` метрики не пишутся в файл; часть идёт в stdout через свои флаги.

Быстрый поиск в коде: `rg "METRICS:" src`

---

## Файлы метрик

| Файл | Назначение |
|---|---|
| `queue-list.txt` | Каждое добавление в очередь: запрос/URL, заголовок, watch URL после разрешения |
| `skips.txt` | Скипы: обычный / `quick_skip` (быстрее `MUSIC_QUICK_SKIP_MS` мс) |
| `session-tracks.txt` | Каждый старт трека: номер в сессии, мс от начала сессии, мс от предыдущего |
| `autoplay-spawn.txt` | JSON на каждую попытку автоплея: `groqTrace`, `allQueries`, `outcome`, трек |
| `baseline.txt` | При `AUTOPLAY_BASELINE_LOG=1`: задержки и счётчики Groq по spawn/idle/play |
| `playability.txt` | При `AUTOPLAY_PLAYABILITY_CACHE_SHADOW=1`: shadow_put / soft_ok по bad URL |
| `bridge.txt` | Подсказки внешнего сервера рекомендаций, ошибки boost/HTTP |
| `autoplay-debug.txt` | При `AUTOPLAY_DEBUG=1`: стадии Groq, stale-отбрасывания, policy |
| `music-ui.txt` | Запись при дописывании автоплей-трека в список Discord |

---

## Поведение при `METRICS_TXT_ENABLED=0`

| Подсистема | Куда идёт |
|---|---|
| Baseline | stdout `[autoplay:baseline]` при `AUTOPLAY_BASELINE_LOG=1` |
| Debug / Groq | stdout `[autoplay:debug]` при `AUTOPLAY_DEBUG=1` |
| Playability shadow | stdout `[playability:cache]` при shadow |
| Bridge | stdout `[bridge] …` |
| Stale guard | stdout `[autoplay:stale]` + debug при `AUTOPLAY_DEBUG=1` |
| Очередь / скипы / сессия / spawn | Ничего (функции — no-op). Включите TXT или смотрите DEBUG/baseline в консоли |

## Консольные логи (текущие)

Диагностические `console.log` добавлены в горячие пути (не за флагом — будут убраны после стабилизации):

| Префикс | Что |
|---|---|
| `[play] ▶` | Начало `runPlayNext`, `src`, url |
| `[play] starting stream (Xms)` | Перед вызовом `streamUrl` |
| `[play] stream started (Xms)` | После `player.play` |
| `[autoplay] spawn start` | До `spawnAutoplayPlaylist` |
| `[autoplay] spawn end elapsed=Xms` | После spawn |
| `[autoplay] groq done elapsed=Xms` | После retrieval |
| `[autoplay] search done elapsed=Xms` | После `runAutoplaySearchStep` |
| `[prefetch] pool_hit=full/fast` | Prefetch пул использован |
| `[prefetch] fast_phase_duration_ms=X` | Время фазы 1 |
| `[btn] customId user=... guild=...` | Нажатие кнопки |

## Цепочка данных

```
enqueue / skip / track_started  → queue-list | skips | session-tracks
spawn автоплея                  → autoplay-spawn + autoplay-debug
baseline hooks                  → baseline.txt
playability shadow              → playability.txt
bridge                          → bridge.txt
UI список                       → music-ui.txt
```

---

## Ключевые метрики производительности

| Метрика | Где читать | Ориентир |
|---|---|---|
| `pool_hit` (full/fast/miss) | Консоль `[prefetch]` | `full` или `fast` — prefetch работает |
| `pool_full_arrived_before_end` | `[prefetch] full_phase_done` в консоли | Показывает достаточность 15 с задержки |
| `fast_phase_duration_ms` | `[prefetch] fast_phase_duration_ms=X` | Должно быть < 5 с |
| `spawn elapsed` | `[autoplay] spawn end elapsed=Xms` | < 10 с при пустом пуле; < 50 мс при pool_hit |
| Quick-skip penalty | `data/metrics/skips.txt` | `quick_skip` → Groq получает это как negative context |
