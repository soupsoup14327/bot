# Статус PAWPAW

**Обновлено:** 2026-04-19  
**Режим:** обкатка и наблюдение за живым поведением

---

## Закрыто

- Data layer из ADR-001 реализован и подключён в runtime.
- Основной dev/test backend — SQLite; для Postgres есть отдельный generated migration pack и зелёный CI smoke-path.
- Лайки работают end-to-end: кнопка ❤ пишет в БД, `/likes` читает персональное избранное.
- Playback history пишет natural finish и skip/previous в `track_plays`.
- Per-user retention ограничивает рост `track_plays`.
- CI matrix SQLite/Postgres зелёная.

## В обкатке

- Живое поведение лайков, `/likes` и playback history в Discord.
- Поведение SQLite-файла и миграций при рестартах.
- Накопление `track_plays` и реальная необходимость более жёсткой retention policy.
- Поведение autoplay в живом трафике: escape уже есть как emergency backstop, а distinct shortlist и artist quarantine выкатываются отдельно и пока оцениваются как candidate-layer улучшения diversity.

## Отложено

- Companion-app и включение HTTP API в production path.
- Advisory lock, `/health/db`, slow-query observability для Postgres path.
- Shared DB connection refactor между runtime facade-модулями.
- Продуктовый recommender, который начнёт использовать уже собираемые likes/history.
- Housekeeping по GitHub Actions warnings, когда дойдёт очередь до инфраструктурной полировки.
