# Переменные окружения

**Обновлено:** 2026-04-18  
**Шаблон:** `.env.example` (корень `bot/`, **авто-генерируется** из реестра)  
**Ваши значения:** `.env` (никогда не коммитить)  
**Единственный источник правды:** `scripts/env-registry.mjs`

---

## Быстрый старт

```bash
npm run env:setup           # создаёт .env с разумными дефолтами
# заполнить DISCORD_TOKEN (обязательно) и GROQ_API_KEY (если нужен ИИ)
npm run env:doctor          # человекочитаемый чеклист + подсказки по исправлению
npm run verify              # полная проверка модулей + ENV + дрейф .env.example
```

**Пресеты (не перезаписывают токены):**
```bash
npm run env:preset:stable        # сбалансированные дефолты
npm run env:preset:conservative  # меньше экспериментального
```

**Регенерация `.env.example`** (после правок реестра):
```bash
npm run env:example          # перезаписать .env.example из реестра
npm run env:example:check    # упасть, если .env.example разошёлся с реестром
```
`env:example:check` прошит в `npm run verify`, так что забыть нельзя.

## Автоплей — стратегия

| Переменная | Дефолт | Описание |
|---|---|---|
| `AUTOPLAY_FAST_LANE_ENABLED` | `0` | Fast path без Groq: детерминированные запросы из последнего трека/seed. Не работает при pivot (доминирующий артист) |

| `AUTOPLAY_RECOVERY_STREAK_MIN` | `2` | Порог плохих spawn для активации recovery |
| `AUTOPLAY_POLICY_ENABLED` | `1` | Политика запросов: анти-луп, quarantine, token-family |

## Валидация

```bash
npm run verify:env   # только ENV-проверки
npm run verify       # модули + ENV
```

Проверяет:
1. Все ключи из `src/*.js` присутствуют в реестре
2. Ключи реестра задокументированы в `.env.example`
3. Если `.env` есть — значения проходят тип/диапазон
4. Внутренние переменные (например, `FFMPEG_PATH`) исключены из операторских проверок
